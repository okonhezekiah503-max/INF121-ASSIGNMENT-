Module 1 - Introduction to HTML

HTML stands for HyperText Markup Language. It serves as the structural backbone for every single resource deployed across the World Wide Web.

Essential Concept Summary

An HTML document is fundamentally a plain text file encoded with a .html file extension. By utilizing predefined markup tags, developers dictate how a web browser parses and visually renders text, media content, inputs, and semantic sections.

Core Document Layout Tags

HTML Tag Technical Purpose
<html> The absolute root element enclosing all content within the document.
<head> Contains crucial document metadata, character sets, external script linkages, and CSS titles.
<body> Encapsulates all user-facing, visible content elements rendered in the browser viewport.
<h1> to <h6> Defines hierarchical text headings, with h1 representing the highest structural importance.
<p> Encloses discrete block-level blocks of text formatted as paragraphs.
<img> An empty/self-closing element used to embed image resources natively into the canvas.

Module 2 - HTML Formatting and Lists

HTML natively provides distinct inline tags engineered explicitly for manipulating text values, establishing visual hierarchies, and enhancing technical content readability.

Text Formatting Semantics

  • <strong>: Imparts strong structural importance, rendering text in bold.
  • <em>: Emphasizes spoken text contextually, rendering it in italics.
  • <b>: Applies visual bold formatting purely for stylistic distinction without adding structural weight.
  • <i>: Sets text in italics for alternative typographic voice styling.
  • <u>: Applies a clear, continuous underline text decoration style.

Data Lists Structures

Lists help structure unstructured groups or explicit ordered instructions sequentially:
